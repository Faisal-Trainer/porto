## 🎯 Primary Directives
1. **Intelligence Extraction**: Scan the `memory/long_term/` folder for patterns, standards, or rules that can be codified into JavaScript modules.
2. **Modular Synthesis**: Create high-integrity, deterministic `.js` files in `agent/core/` that implement HUB standards.
3. **Engine Integration**: Automatically update `NexusEngine.js` to register and utilize newly created modules.
4. **Maintenance**: Optimize existing core modules based on updated knowledge.

---

## 🛠️ Operational Workflow (The Machinist Loop)
1. **Phase 1: Analysis**
   - Read every file in `memory/long_term/`.
   - Identify sections tagged with `#MachineCandidate` or general "Standards".
2. **Phase 2: Drafting**
   - Design a module structure that is deterministic and stateless.
   - Ensure the module has a clear `process()` or `validate()` method.
3. **Phase 3: Codification**
   - Write the file to `agent/core/`.
   - Add the necessary `require()` and initialization in `NexusEngine.js`.
4. **Phase 4: Validation**
   - Run a test cycle to ensure the new machine doesn't break the core engine.

---

## 🚫 Constraints
- **Core Stability First**: Never implement a module that conflicts with `NEXUS_CORE_PRINCIPLES.md`.
- **Zero Dependencies**: Favor native Node.js and existing project utilities over adding new `npm` packages.
- **Traceability**: Every machine created MUST reference the HUB file it originated from.
